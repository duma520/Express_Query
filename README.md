### **3. 第二部分：零基础入门指南**

如果您是第一次使用本软件，请跟随本部分内容，一步步上手。

#### **3.1 我为什么要用这个软件？**

*   **一站式管理：** 您不再需要打开淘宝看圆通、打开京东看顺丰。在这里，一个界面管理您所有的快递。
*   **清晰明了：** 通过智能分类（待揽件、在途中、已到驿站等），您一眼就能找到您最关心的包裹。
*   **私密安全：** 您的所有快递数据都存储在您自己的电脑上，我们**不会**上传任何信息到任何服务器。您的隐私，只有您自己掌握。
*   **完全免费：** 本软件基于GNU AGPL v3.0协议开源，永久免费使用。

#### **3.2 第一次使用：安装与启动**

本软件为绿色版，无需安装。
1.  将 `Express_Query.exe` 文件解压到您喜欢的任意目录（例如 `D:\我的软件\快递查询`）。
2.  双击 `Express_Query.exe` 即可运行。

**首次启动与用户系统：**
3.  启动后，您会看到一个 **“选择用户”** 的对话框。
    *   **什么是用户系统？** 这是为了在一台电脑上，供多个家人或同事使用而设计的。每个用户拥有独立的数据库，互不干扰。
    *   **默认用户：** 系统会自动创建一个名为 `默认用户` 的账号。您可以直接选中它，点击 `登录` 即可开始使用。
    *   **添加自己的用户：** 点击 `添加用户`，输入您想要的名字（例如“张三”），然后选中它登录。您也可以在此界面修改或删除用户。

#### **3.3 认识一下我们的界面（马卡龙色系）**

登录后，您将看到主界面。它由以下几部分组成：

1.  **顶部工具栏：** 显示当前用户和API账号，还有全局搜索框和几个功能按钮。
2.  **主标签页：** 这是核心功能区，包括：
    *   **📦 快递汇总：** 您所有快递的“大本营”，在这里进行管理。
    *   **🔍 快递查询：** 添加新快递的地方。
    *   **📋 查询历史：** 您所有查询过的单号记录。
    *   **📊 配额信息：** 查看API调用次数。
    *   **🗄️ 数据库管理：** 数据备份与恢复。
3.  **状态栏：** 显示软件的运行状态。

> **色彩说明：** 您会注意到我们的软件色彩非常柔和。这是特意为您设计的**马卡龙色系**，旨在减少长时间面对屏幕的视觉疲劳。更巧妙的是，不同的快递状态分类（如待揽件、已签收）会有不同的颜色背景，方便您快速区分。

#### **3.4 完成我的第一次查询（简单模式）**

1.  切换到 **“🔍 快递查询”** 标签页。
2.  在 **“快递单号”** 输入框中，输入您的快递单号（例如 `YT1234567890`）。
3.  **“快递公司”** 选择 `自动识别` 即可，系统会智能判断。
4.  点击 **“开始查询”** 按钮。
5.  稍等片刻，如果查询成功，会弹出提示框，并显示快递的物流轨迹。
6.  系统会自动询问您是否切换到“快递汇总”查看，点击 `是`。

**常见情况：**
*   **查询成功但提示“等待揽件”：** 这说明商家已经填了单号，但快递员还没取走。这是正常现象，软件会将其归类到“⏳ 待揽件”。
*   **查询失败：** 可能是网络问题，或者单号输入错误。请检查后重试。

#### **3.5 管理我的所有快递（汇总模式）**

这是您最常使用的页面。所有您查询过的快递都会自动添加到此处，并按状态进行分类。

*   **分类标题栏：** 每个分类（如“✈️ 在途中”）右侧都有 `(数字)` 显示该分类下的快递数量。
*   **快递卡片：** 每个快递都显示为一个独立的卡片，包含了单号、公司、状态、更新时间，甚至还有预估送达时间。
*   **快捷操作：**
    *   **点击备注区域：** 可以给快递添加备注，比如“买的黑色手机壳”。
    *   **点击最新轨迹：** 可以查看该快递的完整物流详情。
    *   **右侧的“刷新”按钮：** 手动更新该快递的最新状态。
    *   **右侧的“删除”按钮：** 将已签收或不再关心的快递从汇总中移除。

### **4. 第三部分：进阶功能详解**

当您熟悉了基本操作后，以下功能将极大提升您的效率。

#### **4.1 快递详情页：一切尽在掌握**

在快递汇总页面，点击任何一个快递卡片上的 **最新物流轨迹** 区域，就会弹出详情对话框。

*   **基本信息：** 清晰列出单号、公司、状态和重要的时间节点。
*   **完整轨迹：** 按时间倒序排列，最新的物流状态在最上面。
*   **主动刷新（F5）：** 如果您迫切想知道某个包裹的最新动态，可以在详情页里直接按 `F5` 或点击 `刷新` 按钮，系统将立刻向服务器查询最新状态，而无需刷新整个汇总列表。

#### **4.2 智能分类系统：让一切井井有条**

这是本软件的核心亮点之一。系统不仅仅依赖快递公司的状态码，还会**分析物流轨迹的文字内容**，进行智能归类。

| 分类图标 | 分类名称 | 判定逻辑（简化说明） | 背景色（马卡龙） |
| :--- | :--- | :--- | :--- |
| ✅ | 已签收 | 状态码为`3`或文字包含“已签收”、“妥投”等 | 薄荷绿 |
| 📍 | 已到驿站 | 文字包含“菜鸟”、“丰巢”、“取件码”等关键词 | 蜜桃橙 |
| 🚚 | 派件中 | 状态码为`5`或文字包含“派送中”、“派件中” | 天空蓝 |
| ✈️ | 在途中 | 状态码为`0`或文字包含“运输中”、“已发出” | 薰衣草紫 |
| 📮 | 已揽收 | 状态码为`1`或文字包含“已揽件”、“已揽收” | 苹果绿 |
| ⏳ | 待揽件 | 刚添加单号，尚无任何物流信息 | 棉花粉 |
| ⚠️ | 疑难件 | 状态码为`2`或文字包含“疑难”、“异常” | 珊瑚红 |
| ↩️ | 退件 | 状态码为`4`或`6`，或文字包含“退回” | 摩卡 |
| 🛃 | 清关中 | 文字包含“海关”、“清关” | 柠檬黄 |
| 📦 | 其他 | 无法归入以上分类的快递 | 奶油白 |

> **专业提示：** 这种“状态码+语义分析”的双重判断机制，比单纯依赖状态码的软件要**准确得多**。例如，有些快递在放入丰巢柜后，状态码可能还是“派件中”，但我们通过分析“您的快递已存放至丰巢柜”的文字，就能准确将其归类到“已到驿站”，让您第一时间知道可以去取件了。

#### **4.3 全局搜索与历史记录**

*   **全局搜索 (Ctrl+F)：** 无论您在哪个页面，按下 `Ctrl+F`，输入单号或公司名，系统会自动跳转到“查询历史”页面并展示搜索结果。这是快速定位某个快递的神器。
*   **查询历史：** 这个页面记录了您所有**查询过**的单号（无论是否添加到汇总）。您可以在这里找到很久以前查过的某个单号，并一键“重新查询”。历史记录也支持导出为CSV文件。

#### **4.4 个性化备注与截图**

*   **备注：** 点击快递卡片的备注区域，在弹出的对话框里，您可以用文字记录下这个快递是什么商品。我们甚至提供了“衣服”、“电子产品”等快捷备注按钮。这对于一次性购买多件商品的用户来说，是区分包裹的利器。**快捷键：** 在备注编辑框内按 `Ctrl+Enter` 可以快速保存。
*   **截图：** 点击快递卡片左侧的图片区域，您可以通过菜单选择：
    *   **选择文件添加：** 从电脑里选择一张图片（比如商品订单截图）。
    *   **从剪贴板粘贴：** 如果您已经用微信或QQ截了图，直接用 `Ctrl+V` 或选择此选项即可将截图保存到快递条目中。这对于记录商品信息、订单纠纷凭证等非常有用。
    *   **查看大图/删除截图。**

#### **4.5 数据导出：我的数据我做主**

在“快递汇总”和“查询历史”页面，都有 **“导出汇总”** / **“导出”** 按钮。点击后，您可以将当前列表中的所有数据导出为一个 `.csv` 文件。

*   **什么是CSV文件？** 这是一种通用的表格数据格式，可以用Microsoft Excel、WPS表格或任何文本编辑器打开。
*   **用途：** 您可以用它来制作自己的快递统计报表、进行数据分析，或者仅仅是做一个冷备份。

### **5. 第四部分：高级功能与专业应用**

本章节面向需要处理大量快递的电商从业者、小型企业主或高级用户。

#### **5.1 API账号管理：突破查询限制**

**背景知识：** 快递查询服务由“快递100”等第三方API提供商提供。免费账号通常有每日查询次数限制（例如100次/天）。对于个人用户足够，但对于业务量大的用户则捉襟见肘。

**解决方案：** 本软件支持**多API账号管理**和**智能额度切换**。
1.  您可以去快递100开放平台注册多个免费账号。
2.  在本软件的 `设置` -> `API账号管理` 中，添加这些账号的 `Customer ID` 和 `Auth Key`。
3.  启用 **“启用额度用完自动切换账号”**。
4.  **工作流程：** 当主账号今日查询次数达到上限后，系统会**自动无缝切换到**下一个可用的备用账号进行查询。您几乎感知不到这个过程，从而理论上获得了 `N * 100` 次/天的查询能力。

> **专业提示：** 在账号管理界面，您可以通过“上移”、“下移”按钮调整账号的优先级。每个账号的今日已用/总额度一目了然。您还可以手动“重置额度”（注意：这只是重置本地的计数，并不会重置服务器端的真实配额，主要用于管理）。

#### **5.2 导入/导出账号配置**

如果您需要将配置好的多套API账号分享给同事，或者进行备份：
*   **导出账号：** 在账号管理界面，点击“导出账号”。您可以选择**导出完整密钥**（用于备份）或**部分隐藏密钥**（用于安全分享）。
*   **导入账号：** 在另一台电脑上，点击“导入账号”，选择之前导出的JSON文件即可。您可以选择**替换现有账号**或**合并**。

#### **5.3 智能送达时间预估（时效数据）**

**功能原理：** 这是一个**基于本地历史数据的机器学习（简化）** 应用。
1.  **数据收集：** 每当您的一个快递被签收，系统会自动记录下它的揽收时间、签收时间，并计算出运输耗时。同时会记录下快递公司（如“中通”）。
2.  **模型训练：** 随着您使用时间增长，本地数据库会积累大量的历史时效数据。
3.  **智能预估：** 当您查询一个新的、还在运输途中的快递时，系统会查询**同一家快递公司**的历史平均时效，来预估这个新快递大概还需要多久送达。

*   **举例：** 您已经在本软件里签收了20个中通快递，系统计算出中通从揽收到签收平均耗时2.3天。现在您有一个新的中通快递，刚被揽收，系统就会提示：“预计2.3天送达 (基于20条历史记录)”。
*   **管理时效数据：** 在 `设置` -> `时效数据管理` 中，您可以查看当前已记录的数据量，也可以“清空时效历史数据”让系统重新学习。

> **专业解读：** 这种本地化的预估方式，其准确性会随着您个人数据的积累而**越来越高**，因为它反映的是**您所在区域的、您经常使用的快递公司的**真实表现，比任何通用预估都更贴近您的实际情况。同时，所有计算都在本地完成，**数据隐私得到绝对保护**。

#### **5.4 数据库备份与恢复**

您的所有快递数据都存储在本地的 `data` 文件夹下的 `user_xxx.db` 数据库文件中。为防止数据丢失，我们提供了强大的备份管理功能。

*   **自动备份：** 系统默认**每小时**会自动创建一个备份。
*   **手动备份：** 在 `数据库管理` 标签页，点击 `备份管理` -> `立即备份`。
*   **恢复备份：** 在备份列表中，选择一个时间点的备份，点击“恢复选中备份”。恢复前系统会自动创建当前状态的“回滚备份”，确保您可以随时反悔。
*   **备份设置：** 您可以设置最大备份数量（默认30个），超过数量的旧备份会自动删除，以节省磁盘空间。

#### **5.5 多用户系统与配额监控**

*   **多用户系统：** 如前所述，每个Windows用户都可以创建自己的软件用户，数据完全隔离。这对于家庭或办公室共用电脑的场景非常实用。
*   **配额信息标签页：** 这里实时展示了当前使用的API账号的每日总配额、今日已用和剩余次数。进度条会随着使用次数增加而变化，在接近上限时会变色警告。您可以点击“刷新配额信息”来手动获取最新的服务器端数据。

### **7. 第六部分：常见问题解答 (FAQ)**

**Q1: 为什么我查询一个单号后，在汇总里看不到？**
A1: 请首先检查您的 `快递汇总` 页面中的各个**分类是否处于折叠状态**。点击分类标题栏的 `▶` 按钮即可展开。如果展开后仍没有，请检查是否是网络问题导致添加失败。

**Q2: “待揽件”状态会持续多久？**
A2: 从商家填单到快递员取走包裹，中间可能有数小时到一天的时间差。您可以点击快递卡片上的“刷新”按钮手动更新，或者等待系统在您下次刷新时自动更新。

**Q3: 我可以把软件和数据放在U盘里随身携带吗？**
A3: **完全可以。** 本软件为纯绿色便携软件，所有数据和配置都保存在软件目录下的 `data` 文件夹中。将整个软件目录复制到U盘，即可在任何电脑上运行，您的所有数据都会原封不动地带在身边。

**Q4: 更换电脑后，如何迁移我的数据？**
A4: 将旧电脑上软件目录下的 `data` 文件夹，完整地复制到新电脑的软件目录下，覆盖即可。

**Q5: 截图功能支持哪些格式？为什么我的图片很大？**
A5: 支持 `PNG, JPG, JPEG, BMP, GIF` 等常见格式。为了平衡数据库大小和图片质量，软件在存储前会对图片进行适度压缩。

**Q6: API额度用完了会怎样？还能查吗？**
A6: 如果启用了自动切换且有可用备用账号，系统会自动切换。如果所有账号额度都耗尽，您将无法查询新的单号，但刷新已有快递的状态和查看历史数据不受影响。

**Q7: 这个软件会偷我的数据吗？**
A7: **绝对不会。** 本软件的所有代码都是开源的（[GitHub链接](https://github.com/duma520/Express_Query)），任何人都可以审查代码。它除了向您配置的API服务商（如快递100）发送单号查询请求外，不会向任何其他服务器发送任何数据。您的所有信息都只存储在您自己的电脑上。
