### 第一部分：新手入门 - 从零开始查快递

这部分是为完全没有使用过本软件的用户准备的。即使您对电脑操作不太熟悉，只要跟着步骤走，也能轻松掌握。

#### 1.1 软件是什么？能做什么？

简单来说，`Express_Query` 是一个**超级快递查询和管理工具**。您可以把它想象成一个您个人或团队的“**物流数据指挥中心**”。

- **对普通用户来说，它能：**
    - 在一个地方查询几乎所有主流快递公司的包裹。
    - 自动保存您查过的所有快递，形成一个清晰的列表。
    - 智能地将您的快递分成“派件中”、“已签收”、“在途中”等类别，一目了然。
    - 给您的快递添加备注（比如“买的黑色卫衣”），这样就不会弄混了。

- **对专业用户（电商、物流）来说，它能：**
    - **批量导入**上百个快递单号，一次性查询和监控。
    - **批量刷新**所有在途快递，自动获取最新状态。
    - 通过**多用户系统**，让不同员工拥有独立的数据空间。
    - **智能API账号管理**，当一个查询额度用完后，自动切换到下一个，保证业务不中断。
    - **物流节点时间分析**：它会学习您过往的快递数据，告诉您“从A地到B地大概需要多久”、“您的包裹在某个分拣中心通常会停留几个小时”。
    - **送达时间预估**：基于真实的历史签收数据，对当前在途的快递给出更精准的送达时间预估。

#### 1.2 第一步：安装与首次启动

`Express_Query` 是绿色软件，无需安装。

1. **下载：** 从官方渠道（如GitHub Releases页面）获取软件压缩包。
2. **解压：** 将压缩包解压到您电脑上的任意文件夹（例如 `D:\Express_Query`）。
3. **运行：** 进入解压后的文件夹，找到名为 `Express_Query_Pro.exe`（或类似名称）的文件，**双击运行它**。

> **💡 小提示：**
> 首次运行时，您的杀毒软件可能会弹出警告。这是因为本软件是开源且未经过商业代码签名的。请放心，这是正常的，选择“允许运行”或“信任”即可。

#### 1.3 第二步：认识用户系统

软件启动后，您首先看到的是一个“**选择用户**”的登录对话框。这是本软件的核心特性之一。

- **“用户”是什么？** 您可以把一个“用户”看作一个**独立的数据空间**。不同用户之间的快递查询历史、汇总数据、API账号设置等都是完全隔离、互不影响的。
- **为什么要用？**
    - **个人使用：** 您只需一个“默认用户”就足够了。
    - **家庭共用：** 可以为您和您的家人分别创建用户，这样大家查的快递就不会混在一起。
    - **团队使用：** 公司的客服A、客服B、物流专员C都可以拥有自己的账号，各自管理自己负责的订单，数据清晰，责任明确。

- **如何操作？**
    1. 首次启动，系统会自动为您创建一个名为“**默认用户**”的账号。
    2. 选中“默认用户”，点击右下角的“**登录**”按钮即可进入主界面。
    3. 您也可以点击左下角的“**添加用户**”，输入一个新的名字（如“张三”），然后登录这个新账号。

#### 1.4 第三步：第一次查询快递

进入软件主界面后，您会看到一个包含多个标签页的窗口。请确保您当前在“**🔍 快递查询**”标签页。

1. **输入单号：** 在“**快递单号：**”后面的输入框中，输入您要查询的快递单号。例如：`YT1234567890123`。
2. **选择公司：** 在“**快递公司：**”后面的下拉菜单中，选择对应的快递公司。如果您不确定是哪家公司，请保持默认的“**自动识别**”。
3. **开始查询：** 点击右侧的“**🔍 开始查询**”按钮。

> **🎯 举例说明：**
> 您在淘宝上买了一件衣服，卖家通过**圆通速递**发货，给了您一个单号：`YT9876543210123`。
> - 您将这个单号输入到软件中。
> - 快递公司选择“自动识别”或“圆通速递”。
> - 点击查询，软件就会立刻去圆通的服务器获取这个包裹的最新物流信息。

#### 1.5 第四步：理解查询结果

查询成功后，下方区域会显示查询结果，它由三个子标签页组成：

- **📋 基本信息：** 以表格形式列出关键信息，如快递单号、快递公司、当前状态、是否签收等。
- **📜 物流轨迹：** 这是您最关心的部分。它按时间倒序（最新的在最上面）显示了包裹经历的每一个环节。
    - `【2026-04-15 09:30:25】 您的快件已签收，签收人：本人。`
    - `【2026-04-15 08:15:10】 您的快件正在由快递员派送中，快递员：李师傅，电话：138XXXXXX。`
    - `【2026-04-14 22:10:05】 您的快件已到达【上海转运中心】。`
- **📄 原始数据：** 这里显示的是快递公司服务器返回的原始JSON数据，主要供开发人员或技术人员分析使用，普通用户可以忽略。

> **💡 小提示：**
> 如果查询后显示“**暂无物流信息，等待揽件**”，这代表单号是有效的，但快递员还没有从卖家那里取走包裹。这是正常现象，过半天或一天再刷新就会有信息了。

#### 1.6 第五步：将快递加入“我的关注”

在查询成功后，软件会自动将这个快递添加到您的“**📦 快递汇总**”页面。这是本软件最核心的功能之一。

现在，请点击顶部的“**📦 快递汇总**”标签页。您会发现，刚刚查询的快递已经以一个漂亮的“卡片”形式出现在这里了。从此以后，您就可以在这里集中管理所有您关心的快递了。

### 第二部分：日常使用 - 管理您的所有快递

这部分将详细介绍您每天都会用到的核心功能，帮助您高效地管理所有快递。

#### 2.1 “快递汇总”页面详解 - 您的物流仪表盘

“快递汇总”页面是您使用本软件时停留时间最长的地方。它的设计理念就是一个**清晰、直观的物流仪表盘**。

- **卡片式布局：** 每一个快递都以一个独立的卡片展示。
    - **左侧：** 图片区域，您可以添加截图（比如商品图片、订单截图）。
    - **中间：**
        - **第一行（加粗）：** 快递单号、快递公司、当前状态、添加/更新时间，以及**智能预估的送达时间**。
        - **第二行：** 备注信息，您自己添加的描述。
        - **第三行：** 最新的一条物流轨迹。
    - **右侧：** “刷新”和“删除”按钮。

#### 2.2 强大的自动分类功能

为了不让您的快递列表杂乱无章，本软件会自动对您的所有快递进行**智能分类**。分类的依据是快递的状态码和物流轨迹内容。

您会在“快递汇总”页面看到以下可折叠的分类组：

- **✅ 已签收：** 所有已经完成签收的包裹。
- **📍 已到驿站：** 已经到达菜鸟驿站、丰巢柜等代收点，等待您去取件的包裹。
- **🚚 派件中：** 正在由快递员进行派送的包裹。
- **✈️ 在途中：** 正在运输途中的包裹。
- **📮 已揽收：** 快递员已取件。
- **⏳ 待揽件：** 单号已录入，但快递员还未取件。
- **⚠️ 疑难件：** 出现异常情况的包裹（如地址不详、电话不通）。
- **↩️ 退件：** 正在退回给寄件人的包裹。
- **🛃 清关中：** 正在海关清关的国际包裹。
- **📦 其他：** 无法归入以上分类的包裹。

您可以点击每个分类组标题右侧的 **▼** 按钮来**折叠或展开**该分类下的所有快递卡片，让界面保持整洁。

#### 2.3 手动添加快递与批量导入

除了通过“查询”页面添加，您还可以直接在这里添加。

- **手动添加（适用于单个单号）：**
    1. 在“快递汇总”页面的顶部工具栏，有一个“**📝 手动添加单号:**”输入框。
    2. 输入单号，选择快递公司，点击“**➕ 添加到汇总**”按钮即可。软件会立即在后台查询该单号并添加到列表中。

- **批量导入（适用于大量单号）：** 这是为电商卖家或物流专员设计的强大功能。
    1. 点击顶部工具栏的“**📦 批量导入**”按钮。
    2. 在弹出的对话框中，您有三种方式导入单号：
        - **从文件导入：** 选择一个 `.txt` 或 `.csv` 文件，文件中每行一个快递单号。
        - **粘贴多行文本：** 直接将Excel表格中的一列单号复制粘贴进来。
        - **手动输入：** 在下方文本框中，每行输入一个单号。
    3. （可选）您可以选择一个默认的快递公司，或者启用“**自动识别**”让系统智能判断。
    4. 点击“**解析单号**”，系统会预览并去重，告诉您有多少个有效单号。
    5. 点击“**开始导入**”，系统会逐个查询并添加到您的汇总列表中。

#### 2.4 为快递添加备注与截图

这是一个非常人性化的功能，能极大提升管理效率。

- **添加备注：**
    - 在任意快递卡片上，点击中间的“**点击添加备注...**”按钮。
    - 在弹出的对话框中，输入您想备注的内容，例如“**给儿子买的蓝色运动鞋**”、“**李女士的退货**”、“**发票**”。
    - 您还可以使用“快捷备注”功能，一键添加“衣服”、“电子产品”等标签。
    - 添加后，备注信息会清晰显示在卡片上，方便您快速识别。

- **添加截图：**
    - 在任意快递卡片上，点击最左侧的图片区域。
    - 您可以通过右键菜单，选择：
        - **从文件选择...**：上传商品图片或订单截图。
        - **从剪贴板粘贴**：如果您用微信/QQ截了图，可以直接粘贴。
        - **从URL下载...**：直接输入图片的网络地址。
    - 上传后，图片会显示在卡片上，点击可以查看大图。

#### 2.5 刷新与删除快递

- **刷新：** 快递的状态是动态变化的。您可以点击快递卡片右侧的“**刷新**”按钮，手动获取该单号的最新物流信息。您也可以使用稍后介绍的“批量刷新”功能。

#### 2.6 全局搜索：快速定位您的包裹

当您的快递汇总列表里有几十上百个快递时，找一个特定的包裹可能会变得困难。这时，“全局搜索”功能就派上了用场。

### 第三部分：进阶功能 - 智能与高效

这部分内容开始涉及更高级的应用技巧，帮助您事半功倍。

#### 3.1 自动识别快递公司

您可能已经注意到，在很多地方我们都可以选择“自动识别”快递公司。这个功能是如何工作的呢？

本软件内置了一个**庞大的快递单号格式规则库**。例如：

- 以 `SF` 开头，且长度大于等于12位的 → **顺丰速运**
- 13位纯数字，且以 `98`、`99` 开头的 → **EMS**
- 以 `JT` 开头的 → **极兔速递**
- 以 `JD` 开头的 → **京东物流**

当您选择“自动识别”时，软件会先用这些规则去匹配，如果匹配成功，就直接使用对应的快递公司代码进行查询，**速度更快，也更省API配额**。如果匹配失败，它才会调用API的智能判断接口。

#### 3.2 批量刷新与分类刷新

当您有大量在途快递时，逐个点击“刷新”是低效的。

- **刷新此类：** 在每个分类组的标题栏右侧，都有一个“**刷新此类**”按钮。点击它，软件会**自动刷新该分类下所有快递**的状态。例如，您可以每天早上点击一下“派件中”分类的刷新按钮，快速了解所有正在派送的包裹是否有更新。
- **刷新全部：** 点击“快递汇总”页面顶部工具栏的“**🔄 刷新全部**”按钮，软件会刷新您**汇总列表中的所有快递**。

这两个功能在底层都使用了**多线程技术**，会并发地向API发送请求，并在界面上显示清晰的进度条，告诉您当前刷新到哪一个了，全程流畅不卡顿。

#### 3.3 批量导出快递单号

如果您需要将系统中的快递单号分享给他人，或者导入到另一个系统，可以使用此功能。

1. 点击顶部工具栏的“**📋 批量导出单号**”按钮。
2. 在弹出的对话框中，您可以筛选、全选、反选您想要导出的快递单号。
3. 选择导出格式：
    - **纯文本 (.txt)**：每行一个单号，最简洁。

    - **JSON文件 (.json)**：结构化的数据格式，方便程序处理。
4. 点击“导出选中”，选择保存位置即可。

#### 3.4 强大的右键快捷菜单

为了提高操作效率，我们在快递卡片的各个区域都集成了功能丰富的**右键快捷菜单**。

- **在备注按钮（第二行）上右键：** 可以复制备注，或一键使用快捷备注。
- **在物流轨迹按钮（第三行）上右键：** 可以复制最新物流信息，或直接在浏览器中搜索该单号。
- **在左侧图片区右键：** 包含添加、编辑（旋转、裁剪）、导出、复制、删除截图等全套图片管理功能。

熟练使用右键菜单，能让您的操作效率翻倍。

#### 3.5 API账号管理与智能切换

本软件依赖第三方（快递100）提供的API接口来查询快递数据，而API接口通常有每日查询次数限制。为了解决这个问题，我们设计了强大的API账号管理系统。

- **在哪里配置？** 点击顶部工具栏的“**⚙️ 设置**” -> 选择“**🔑 API账号**”标签页 -> 点击“**管理API账号**”。

- **如何配置？**
    1. 您需要先从快递100等平台申请获得 `Customer` 和 `Auth Key`。
    2. 在管理界面中，点击“添加账号”，输入一个便于识别的账号名称（如“主账号”），并填入申请到的 `Customer` 和 `Auth Key`。
    3. 您可以添加多个账号。

- **智能切换机制（核心功能）：**
    1. 在“设置”或“账号管理”界面中，确保“**启用额度用完自动切换账号**”选项已勾选。

    3. **最近使用优先（智能排序）：** 如果您没有勾选“固定排序”，系统会自动将**最近成功使用过的账号移动到列表第一位**。这意味着，系统会优先尝试使用那个最“健康”、最可能还有额度的账号，大大减少了因额度不足而查询失败的概率。

### 第四部分：专业应用 - 物流数据分析

这部分是 `Express_Query` 区别于普通快递查询软件的核心价值所在，它将您的查询历史转化为有价值的商业洞察。

#### 4.1 物流节点时间分析器 (Logistics Node Analyzer)

这是一个**自学习的物流时间分析引擎**。它通过分析您已经签收或正在运输中的快递轨迹数据，来学习“经验”。

**它做了什么？**
1. **解析轨迹：** 每当有一个新快递（无论是手动查询还是批量导入）进入系统，它都会在后台默默地解析其物流轨迹。
2. **提取节点：** 它会从“【北京转运中心】已发出”、“已到达【上海XX公司】”等文本中，智能提取出**站点名称、城市、动作类型（到达/离开/派送）**。
3. **计算时间：** 它会计算两个节点之间的耗时（运输时间），以及在同一个站点的停留时间。
4. **存入历史：** 这些宝贵的时间数据会被存入 `node_transit_history` 表中，成为后续预估的基础。

**对您有什么用？**
- **在“快递详情” -> “📍 站点时间轴”标签页中，您可以清晰地看到：**
    - 您当前的包裹经过了哪些站点。
    - 在每个站点停留了多长时间。
    - **更强大的是：** 下方“后续站点预估”区域，会基于您的历史数据，告诉您这个包裹**大概还有多久能到达下一个关键节点**（例如“预计 4.5 小时后到达下一转运中心”）。这个预估**不仅限于同公司数据，当同公司样本不足时，还会参考其他快递公司的线路数据**，并明确标注“跨公司参考”，让预估更全面、更可靠。

#### 4.2 送达时间预估器 (Delivery Time Estimator)

这是一个更宏观的预估工具，它不关心中间节点，只关心**从揽收到签收（或到驿站）一共花了多少天**。

- **数据来源：** 它会自动记录您系统中所有**已签收**或**已到达驿站**的快递，提取出始发城市、目的城市、揽收时间、签收/到达时间，并计算出总耗时（天数）。
- **如何体现：**
    - 在“快递汇总”页面的快递卡片上，对于在途的包裹，您会看到类似“**预计04月20日送达**”的提示。这就是基于该快递公司历史时效数据的预估。
    - 点击提示文本，还可以看到更详细的说明，例如“**基于35条该快递公司历史记录，平均2.5天送达**”。

#### 4.3 历史数据重建：让数据说话

如果您是从旧版本升级，或者希望用现有的所有快递数据来“训练”这个分析引擎，可以使用此功能。

1. 进入“**🗄️ 数据库管理**”标签页。
2. 在“历史数据重建”区域，您会看到三个按钮：
    - **📊 重建时效历史数据：** 扫描您现有的所有快递，重新解析并填充 `delivery_history` 表。
    - **📍 重建节点运输历史数据：** 扫描您现有的所有快递，重新解析并填充 `node_transit_history` 表。
    - **🔄 全部重建（时效+节点）：** 同时执行以上两个操作。

> **⚠️ 注意：** 重建操作会清空现有的历史分析数据并用新解析的数据覆盖。这是一个相对耗时的操作（取决于您的快递数量），建议在软件空闲时进行。

#### 4.4 快递详情对话框：信息的集大成者

在任意快递卡片上点击物流轨迹区域，或使用右键菜单中的“查看完整轨迹”，会弹出“快递详情对话框”。这是获取单个包裹最全面信息的地方。

- **📋 基本信息：** 整合了所有关键字段和预估送达时间。
- **📜 物流轨迹：** 完整的、格式化的物流文本。
- **📍 站点时间轴：** （核心）以表格形式清晰展示每个站点的到达、离开和停留时长，并附带后续站点预估。
- **📊 详细分析：** 一个纯文本但结构清晰的“物流节点时间分析报告”，它整合了所有分析结果，包括同公司预估、跨公司参考和最终预估，方便您复制分享。

### 第六部分：版本更新日志与说明

这里将为您详细介绍从 v1.1.54 到 v1.1.71 的版本演进。

#### 6.1 版本 v1.1.71 (最新版) 更新内容

**发布日期：** 2026-04-12 (构建)
**版本号：** `1.1.71`

这是一个**里程碑式**的版本，在用户体验、数据分析和智能化方面进行了**全面、深度的重构与增强**。

**🚀 核心功能与体验升级：**
1.  **用户体验革命：弹窗消息全面转向状态栏提示**
    - **问题：** 旧版本中，大量的操作反馈（如“复制成功”、“添加成功”）会以模态对话框（`QMessageBox`）的形式弹出，用户必须点击“确定”才能继续，严重打断操作流。

2.  **智能分类算法重大优化 (v2.0)**
    - **问题：** 旧版本的分类逻辑在某些边界情况下不准确，例如将“请签收”误判为“已签收”。
    - **改进：** 重构了 `get_state_category` 方法，制定了更严格的优先级判断逻辑：
        - **优先级1：已签收** - 使用更精确的关键词（如“已签收”、“本人签收”），避免误判。
        - **优先级2：已到驿站/代收点** - 增强关键词库。
        - **优先级3：派件中**
        - **优先级4：待揽件** - 修正了对“状态码0”和“暂无物流信息”的判断。
        - **优先级5：已揽收**
        - ...以此类推。
    - **影响：** 快递分类的**准确率大幅提升**，尤其在“待揽件”、“已揽收”、“已签收”这几个关键状态的区分上更加智能。

3.  **物流节点分析器 (Logistics Node Analyzer) 史诗级增强**
    - **问题：** 旧版本的站点名称提取不够准确，对非标准格式的物流信息（如包含电话、人名的站点）解析能力弱。
    - **改进：**
        - **增强站点名称提取 (`_extract_site_name_enhanced`)：** 全新设计的正则表达式，能精准识别并提取 `广东广州海珠区名粤公司[020-22049077]` 或 `浙江平阳县公司-王龙再（0577-28784350）` 这类复杂格式中的**核心站点名称**。
        - **增强揽收员信息提取：** 能更准确地从物流文本中分离出快递员姓名和电话。
        - **扩展城市库与节点类型：** 城市库扩充至涵盖全国所有地级市及重要县级市；节点类型增加了`port`(港口)、`warehouse`(仓库)、`customs`(海关)等。
        - **扩展动作类型：** 动作类型增加了`loading`(装车)、`unloading`(卸车)、`processing`(分拣)、`in_transit`(运输中)、`takeoff`(起飞)、`landing`(降落)等，**时间轴颗粒度更细**。
    - **影响：** 这是本次更新的**最重磅功能**。站点时间轴的数据质量、准确度和信息丰富度得到了**质的飞跃**。

4.  **站点预估功能：引入跨公司数据参考**
    - **问题：** 旧版本的站点预估只基于同公司的历史数据，当数据量少时无法给出预估。
    - **改进：** `estimate_next_stations` 和 `analyze_and_estimate` 方法重写。当同公司数据样本不足时，系统会自动**查询并参考其他快递公司在相同线路上的历史数据**，并在预估结果中明确标注“**[同公司]**”或“**[跨公司参考]**”，同时显示参考的公司代码和样本量。
    - **影响：** **极大地提升了预估功能的可用性和覆盖面**。即使是很少用的小快递公司，也能通过参考主流快递公司的数据得到有意义的预估。

5.  **API账号管理器智能化升级**
    - **问题：** 旧版本的账号切换逻辑简单，容易导致多个账号的额度被不均匀消耗。
    - **改进：** 增加了“**最近使用优先**”的动态排序机制（可通过“固定排序”选项关闭）。当账号因切换而成功使用时，它会被**自动移动到账号列表的第一位**，使得系统在下一次查询时优先尝试这个健康账号。

6.  **新增批量导入解析增强功能**

#### 7.2 常见问题 (FAQ)

**Q1: 为什么我查询时提示“没该功能权限”或“单号不存在”？**
**A:** 这通常有两个原因：
1. **API账号权限问题：** 您使用的API账号（Customer/Auth Key）没有查询该快递公司的权限。快递100对不同的账号套餐开放了不同范围的快递公司查询权限。
2. **单号未录入/未揽件：** 卖家刚刚填写的单号，系统可能还未同步，快递员也还未取件。建议过几个小时再试。

**Q2: 我有很多快递，软件会变卡吗？**
**A:** 本软件在设计之初就考虑了大数据量场景。快递列表采用动态加载和虚拟滚动技术（通过QScrollArea），即使有上千个快递，界面操作依然流畅。数据库也使用了WAL模式和索引优化，查询效率很高。

**Q3: 自动备份的文件在哪里？会占用很多空间吗？**
**A:** 备份文件默认存储在软件目录下的 `data/backups/` 文件夹中。每个备份文件的大小与您当前使用的数据库大小相当。您可以在“设置”->“备份设置”中限制最大备份数量（默认30个），系统会自动清理旧的备份，防止空间无限增长。

**Q4: 我可以把数据迁移到另一台电脑上吗？**
**A:** 完全可以。您只需要将整个 `data` 文件夹复制到另一台电脑上软件的相同目录下即可。`data` 文件夹包含了所有用户的数据、设置和备份。

**Q5: “节点历史数据重建”功能有什么用？我什么时候需要用？**
**A:** 这个功能是用来“训练”物流节点时间分析器的。当您：
- 从旧版本升级到v1.1.71或更高版本，希望用新版更强大的解析引擎来重新分析您已有的历史数据。
- 导入了一大批过去的快递数据（例如从其他系统迁移过来）。
这时，执行一次“全部重建”可以让您的分析数据更准确、更全面。日常使用中，系统会自动记录新的快递，无需频繁手动重建。

**Q6: 我可以自己修改这个软件的代码吗？**
**A:** **当然可以！** 本软件是开源的，遵循AGPL-3.0协议。您可以自由地查看、修改和分发代码，但您修改后的版本也必须以相同的协议开源。

#### 7.3 名词解释

- **API (Application Programming Interface):** 应用程序编程接口。在这里特指快递100提供的查询接口，软件通过它来获取物流信息。
- **API配额 (Quota):** 服务商允许单个账号在一定时间内（通常是每天）调用API的最大次数。
- **WAL (Write-Ahead Logging):** SQLite的一种模式，通过预写日志来提高数据库的并发读写性能和稳定性。
- **物流节点 (Logistics Node):** 物流轨迹中的每一个关键环节，如“已揽收”、“到达XX转运中心”、“离开XX转运中心”、“派送中”、“已签收”。
- **正则表达式 (Regex):** 一种强大的文本模式匹配工具，软件用它来从复杂的物流文本中精确提取城市、站点、电话等信息。
