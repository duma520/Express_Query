### 前言：这是一款什么样的软件？

欢迎使用“快递查询系统”（Express_Query）。这是一款专为个人和中小企业设计的桌面端快递管理工具。

**用一句话概括：** 它能帮你批量管理所有快递单号，自动查询物流状态，并用漂亮的颜色分类显示，让你对每个快递的“旅程”一目了然。

**核心解决痛点：**
1.  **单号太多，记不住？** -> 软件帮你永久保存，随时查看。
2.  **挨个查物流，太麻烦？** -> 一键“刷新全部”，所有快递最新状态尽收眼底。
3.  **快递到哪了，说不清？** -> 自动将快递分为“派件中”、“已到驿站”、“已签收”等，状态清晰。
4.  **怕API额度用完？** -> 支持多账号自动切换，用完一个自动换下一个，查询不断线。

无论您是经常网购的普通用户、管理众多订单的电商卖家，还是负责公司行政采购的专员，这款软件都能成为您的得力助手。

### 第一部分：零基础快速入门（5分钟上手）

这部分是为完全没有编程或软件使用经验的用户准备的。我们将用最简单的语言和例子，教您如何从下载到使用。

#### 1. 准备环境（一次性的工作）

软件是基于Python开发的，但我们已为您打包成`.exe`可执行文件（在Windows系统上），您无需安装任何编程环境。
-   **下载：** 从GitHub的“Releases”页面下载最新的`.exe`文件。
-   **运行：** 双击运行即可。软件会在同目录下自动创建一个`data`文件夹，用于存放所有数据。

#### 2. 第一次启动（选择用户）

-   **情景举例：** 您和您的家人共用一台电脑，但想分开管理各自的快递。
-   **操作：**
    1.  启动软件后，会弹出一个“选择用户”窗口。
    2.  默认有一个“默认用户”，您可以直接选中它，点击“登录”。
    3.  **想新建一个自己的专属账户？** 点击“添加用户”，输入您的名字（例如“张三”），然后登录。
    4.  **简单理解：** 这就像手机的“多用户模式”，每个用户的快递数据是完全隔离、互不干扰的。

#### 3. 添加您的第一个快递（三种最常用的方法）

-   **方法一：手动添加（最直接）**
    -   **情景举例：** 您刚在淘宝买了一个手机壳，商家给了您一个快递单号“1234567890”。
    -   **操作：**
        1.  在主窗口的“快递汇总”标签页，找到“📝 手动添加单号”输入框。
        2.  输入单号 `1234567890`。
        3.  在下拉框中选择快递公司（例如“圆通速递”）。如果不知道，就选“自动识别”。
        4.  点击旁边的“➕ 添加到汇总”按钮。
    -   **结果：** 软件会自动联网查询，如果单号有效，它就会出现在下面的列表里。如果暂时没有物流信息（刚下单，卖家还没发货），它会显示在“⏳ 待揽件”分类中。

-   **方法二：通过查询添加（边查边加）**
    -   **情景举例：** 朋友发来一个单号，您想先看看物流状态，然后再决定是否保存。
    -   **操作：**
        1.  点击“🔍 快递查询”标签页。
        2.  输入单号和快递公司，点击“🔍 开始查询”。
        3.  软件会显示详细的物流轨迹。
        4.  查询成功后，软件会自动询问您是否将本次查询结果添加到汇总中，点击“是”即可。

-   **方法三：批量导入（未来功能）**
    -   **情景举例：** 您开了个小网店，一天有几十个订单需要跟踪。
    -   **操作：** （目前版本需手动添加，但数据结构已为批量导入做好准备，未来更新会支持。目前可单个添加或等待未来更新。）

#### 4. 理解“快递汇总”界面（您的快递控制中心）

添加完快递后，回到“📦 快递汇总”标签页，您会看到快递被分门别类地整理好了。每个快递都显示在一个“卡片”里。

-   **✅ 已签收（绿色边框）：** 快递已成功签收。**举例：** “您的包裹已放在家门口/已由本人签收”。
-   **📍 已到驿站（橙色边框）：** 快递已到达代收点，等待您去取。**举例：** “您的快件已存放在xx小区菜鸟驿站，请凭xxxx取件”。
-   **🚚 派件中（蓝色边框）：** 快递员正在派送。**举例：** “【xx快递】正在派送中，快递员张三，电话138xxxx”。
-   **✈️ 在途中（紫色边框）：** 快递正在运输网络里流转。**举例：** “【xx转运中心】已发出，下一站【xx】”。
-   **⚠️ 疑难件（红色边框）：** 快递出了问题，如地址不详、联系不上收件人等。**举例：** “疑难件，请联系发件人确认地址”。
-   **⏳ 待揽件（粉色边框）：** 商家已下单，但快递员还没取走。**举例：** “暂无物流信息，等待揽件”。
-   **📮 已揽收（浅绿色边框）：** 快递员已取走，开始运输。**举例：** “【xx营业点】的收件员xxx已揽收”。

**在快递卡片上，您可以做什么？**
-   **看状态：** 标题栏清晰地显示了单号、公司、当前状态。
-   **看备注：** 如果给这个快递加了备注（比如“手机壳”），它会显示在卡片上。
-   **看最新轨迹：** 卡片上会显示最新的物流信息摘要。
-   **刷新单个：** 点击卡片右侧的“刷新”按钮，只更新这一个快递的状态。
-   **添加/查看截图：** 点击左侧的图片区域，可以添加或查看快递单的截图（比如订单详情截图）。
-   **删除：** 点击“删除”按钮，可以将此快递从列表中移除。
-   **折叠/展开分类：** 点击每个分类标题栏右侧的“▼”按钮，可以折叠该分类，让界面更清爽。

### 第二部分：专业功能详解（进阶用户指南）

这部分适合需要更高效、更专业管理快递的用户，我们将深入讲解软件的特色功能。

#### 1. API账号管理（核心中的核心）

-   **为什么需要API账号？** 软件本身不提供快递数据，所有查询都通过“快递100”等第三方平台的API接口完成。您需要自己去申请一个免费的API账号（通常每天有100次免费额度）。
-   **在哪里申请？** 百度搜索“快递100 企业版”或“快递100 API”，注册账号后，在“我的API”中就能找到`Customer`和`Auth Key`。
-   **如何配置？**
    1.  点击工具栏的“⚙️ 设置” -> “管理API账号”。
    2.  点击“添加账号”，输入您申请到的`Customer`和`Auth Key`。
    3.  **“智能判断”**：勾选后，如果查询时您忘了选快递公司，软件会根据单号规则自动尝试识别。
    4.  **“每日额度”**：设置一个值，当今日查询次数达到这个值时，该账号会被标记为“额度已满”，软件会自动切换到下一个可用账号。
    5.  **导入/导出：** 如果您有多个账号，可以导出为JSON文件备份，或从备份文件导入。
-   **自动切换逻辑（专业说明）：** 当主账号当天查询次数达到“每日额度”后，再次查询时，软件会按`sort_order`顺序查找第一个`used_today < daily_limit`的账号，并自动切换使用。此功能可通过“启用额度用完自动切换账号”开关控制。

#### 2. 送达时间预估（基于历史数据的智能算法）

-   **工作原理：** 软件会自动记录每个已签收快递的“揽收时间”和“签收时间”，计算出实际送达天数。当您查询一个新的同公司快递时，它会基于这些历史数据，计算平均送达天数，并给出预估日期。
-   **专业说明：**
    -   数据记录在`delivery_history`表中。
    -   预估时会区分“签收”和“到达驿站”两种终点类型，分别计算。
    -   当历史数据少于3条时，会提示“数据较少，仅供参考”。
    -   如果没有历史数据，会使用内置的默认值（如顺丰2天，邮政5天）。
-   **用户价值：** 对于经常发货的商家，可以预估客户何时能收到货，便于安排售后。对于个人，可以大概知道哪天能收到包裹，合理安排时间。

#### 3. 数据库管理（保证数据安全与性能）

-   **WAL模式：** 软件默认开启WAL（Write-Ahead Logging）模式。简单说，这能让你在软件查询时，数据库的读写性能更好，且更不容易损坏。
-   **自动备份：** 软件每小时会自动创建一次数据库备份。备份文件保存在`data/backups`目录下。
-   **手动备份与恢复：**
    -   进入“🗄️ 数据库管理”标签页，点击“💾 备份管理”。
    -   您可以在这里看到所有备份，按时间、类型筛选。
    -   点击“立即备份”可手动创建备份。
    -   选择一个备份，点击“恢复选中备份”，可以将数据库恢复到那个时间点的状态。**（强烈建议在重大操作前手动备份）**
-   **优化数据库：** 长期使用后，数据库文件可能会变大，查询变慢。点击“🔧 优化数据库”会执行`VACUUM`命令，整理磁盘空间，提升性能。

### 第三部分：高级应用与开发指南（面向技术人员）

这部分是为开发者、技术爱好者准备的，说明软件的设计思路、核心代码和如何二次开发。

#### 1. 项目结构与核心模块

-   **`ProjectInfo` 类：** 集中管理所有项目元数据，如版本号、作者、版权信息、项目URL等。修改此处即可全局更新程序信息。
-   **`MacaronColors` 与 `MacaronStyle`：** 定义了马卡龙色系的颜色和Qt样式表（QSS）。开发者可以通过修改这里来轻松更换整个软件的主题皮肤。
-   **`DatabaseManagerPro`：** 数据库操作的核心封装。它使用`sqlite3`模块，并利用线程锁（`Lock`）保证了多线程环境下的数据库操作安全。支持WAL模式。
-   **`UserManagerPro`：** 负责多用户管理。每个用户有独立的数据库文件（`user_{username}.db`），存储在`data/user_data/`目录下，实现了数据隔离。
-   **`ApiAccountManager`：** API账号的核心管理逻辑，包括账号加载、每日额度检查、自动重置和切换。
-   **`DeliveryTimeEstimator`：** 时效预估的核心类。包含城市提取、驿站/签收关键词判断、历史数据查询与平均值计算等逻辑。
-   **`ExpressQueryThreadPro` 与 `BatchRefreshThread`：** 继承自`QThread`的工作线程。所有网络请求都在子线程中执行，防止界面卡顿。
-   **`ExpressQueryProGUI`：** 主窗口类，负责所有UI组件的布局和信号/槽的连接。

#### 2. 核心逻辑解读

-   **快递分类（`get_state_category`）：** 这是软件智能化的关键。它并非仅依赖API返回的`state`状态码，而是结合了`state`和物流信息`context`中的关键词。例如，即使`state`不是3（已签收），只要`context`中包含“签收”二字，也会被归为`signed`类。这种双重判断机制极大地提高了分类的准确性。
-   **自动切换账号（`ApiAccountManager.switch_to_next_account`）：** 实现了一个简单的轮询（Round-Robin）算法。它会从当前账号的下一个开始，依次检查每个账号的`used_today`和`daily_limit`，找到第一个可用的就切换。这保证了查询请求的高可用性。
-   **数据持久化：** 所有快递摘要信息（单号、公司、状态、备注等）都保存在`express_summary`表中，而完整的、未经处理的JSON响应数据则作为`result_data`字段存储。这样做的好处是：一方面可以快速显示摘要信息，另一方面也保留了原始数据，方便日后扩展或重新解析。

#### 3. 如何二次开发与贡献代码

1.  **克隆项目：** `git clone https://github.com/duma520/Express_Query.git`
2.  **安装依赖：** 需要安装`PySide6`和`requests`。推荐使用`pip install -r requirements.txt`。
3.  **环境准备：** 确保您的Python版本为3.8或更高。
4.  **运行与调试：** 直接在IDE中运行`Express_Query.py`即可。设置`DEBUG_MODE = True`可以开启调试输出。
5.  **代码规范：** 项目采用清晰的类结构和命名规范。添加新功能时，请遵循现有模式，例如：
    -   新功能放在新线程中执行，避免阻塞UI。
    -   数据库操作通过`DatabaseManagerPro`进行。
    -   UI字符串尽量放在`ProjectInfo`类中，便于国际化。
6.  **提交Pull Request：** 请先通过GitHub Issues讨论您要修复的问题或要添加的功能，然后再提交代码。

### 附录：常见问题解答（FAQ）

**Q1: 软件是免费的吗？**
A: 是的，本软件基于AGPL v3.0开源协议，完全免费。但调用快递100的API可能需要您自己去申请免费或付费的账号。

**Q2: 为什么我查询时提示“未配置API账号”？**
A: 您需要先点击“设置” -> “管理API账号” -> “添加账号”，输入您从快递100平台申请的`Customer`和`Auth Key`。

**Q3: 软件支持哪些快递公司？**
A: 支持快递100平台下几乎所有主流快递公司，包括顺丰、京东、三通一达、邮政、德邦、极兔等。

**Q4: 我的数据保存在哪里？安全吗？**
A: 所有数据都保存在软件同级目录下的`data`文件夹中。软件不连接任何外部数据库，您的数据完全由您自己掌控，非常安全。

**Q5: 我可以把软件装在U盘里随身携带吗？**
A: 可以。因为数据都是相对路径存储，您可以将整个软件文件夹复制到U盘中，在任何一台Windows电脑上运行，您的数据不会丢失。

**Q6: 软件会自动更新吗？**
A: 目前版本不支持自动更新。当有新版本时，您需要手动从GitHub下载并覆盖旧文件。覆盖前建议备份`data`文件夹。

**Q7: 为什么我的顺丰快递查不到？**
A: 顺丰快递查询通常需要提供收件人手机号后四位。本软件使用的是快递100的公开查询接口，可能不支持此类隐私保护要求高的查询。您可以尝试在快递公司一栏选择“顺丰速运”，并在单号后附加手机号后四位（如“SF1234567890:1234”）进行查询。

**Q8: 如何备份我的所有数据？**
A: 最简单的方法是直接复制整个`data`文件夹。或者，使用软件内置的“备份管理”功能进行备份。

**Q9: 我遇到了软件Bug，如何报告？**
A: 请前往GitHub项目地址，在“Issues”页面创建一个新的问题。请详细描述您的操作步骤、软件版本、操作系统，并提供错误截图或日志（开启调试模式后可以获得）。

**Q10: 如何彻底删除一个用户及其所有数据？**
A: 在登录界面，选中用户，点击“删除用户”。此操作会软删除用户记录，其数据库文件仍保留在`data/user_data/`目录下。如需彻底清理，请手动删除对应的`user_{用户名}.db`文件。
