### **第二部分：从零基础到高级应用完全指南**

本章节是本文档的核心，无论您是何种背景，都可以从中找到适合您的部分。

#### **第一章：新手入门篇（普通网购用户必读）**

本章节面向所有用户，不涉及任何专业术语，确保您能轻松上手。

##### **1.1 软件是用来做什么的？（通俗版）**

想象一下，您在淘宝、京东、拼多多、抖音、快手、小红书等多个平台买了东西，还有朋友寄来的礼物，微商代购发的货。每个快递都有一个长长的单号（比如SF1234567890），您要记住哪个单号是哪家的、买了什么东西、到哪了、签没签收...是不是很头大？

**本软件就是您的“个人快递小管家”！** 它的核心功能是：
1.  **帮您记住所有快递：** 把您所有平台、所有朋友寄来的快递单号一次性或逐个添加进来。
2.  **自动归类整理：** 软件会自动把快递分成“待揽件”、“在途中”、“派件中”、“已签收”等类别，像书架一样摆得整整齐齐。
3.  **一键刷新状态：** 想知道所有快递走到哪了？点一下“刷新全部”，最新的物流状态就全出来了，不用再一个一个去网页上查。
4.  **智能提醒和预估：** 软件会告诉您“这个包裹预计明天送达”、“那个包裹已经在派送了”，甚至能预估从A市到B市大概要多久。

**一句话总结：用了它，查快递、管快递再也不是一件烦心事儿！**

##### **1.2 第一步：软件的安装与启动**

本软件是绿色便携版，无需安装。

1.  **解压文件：** 将您得到的 `Express_Query` 压缩包解压到您喜欢的任何文件夹（例如 `D:\快递查询系统\`）。
2.  **找到程序：** 进入解压后的文件夹，找到名为 `Express_Query_Pro.exe` (或类似名称) 的文件，它就是主程序。
3.  **双击运行：** 双击它，程序就启动了！

**注意：** 程序第一次运行会创建一个 `data` 文件夹，用于存放您的所有数据（快递单号、历史记录、设置等）。请务必保留此文件夹，不要随意删除。

##### **1.3 第二步：首次登录与用户系统**

1.  **选择用户：** 软件启动后会弹出一个“选择用户”的登录框。这是为了方便多人共用一台电脑而设计的，每个人的快递数据是独立存放的。
2.  **默认用户：** 第一次使用时，软件会自动为您创建一个名为“默认用户”的账号。您可以直接双击它登录，或者点击“添加用户”创建一个属于您自己的名字（例如“小明的快递”）。
3.  **登录成功：** 点击“登录”后，您就进入了软件主界面。

**给家庭或公司的建议：** 如果一台电脑多人使用（比如您和家人，或公司前台和同事），可以分别为每个人创建独立用户，这样大家的快递就不会混在一起了。

##### **1.4 第三步：设置API查询功能（必做，否则无法查询！）**

这是最关键的一步。软件需要连接到快递数据平台（使用的是快递100的接口）才能查到物流信息，这需要一个免费的API授权。

1.  **获取API Key：** 请访问 [快递100 API申请页面](https://api.kuaidi100.com/)，按照指引免费注册一个企业账号（个人开发者也可以申请免费套餐），您将获得一个 **Customer ID** 和 **Auth Key**。
2.  **打开设置：** 在软件主界面右上角的工具栏，点击 **“⚙️ 设置”** 按钮。
3.  **管理账号：** 在弹出的“系统设置”窗口中，点击 **“管理API账号”**。
4.  **添加账号：** 在弹出的“API账号管理”窗口中，点击 **“添加账号”**。
    *   **账号名称：** 随便填，例如“我的主账号”。
    *   **Customer：** 粘贴您申请到的 Customer ID。
    *   **Auth Key：** 粘贴您申请到的 Auth Key。
    *   其他选项保持默认即可，点击“保存”。
5.  **关闭窗口：** 依次点击“保存设置”和“关闭”即可。

**名词解释 (针对好奇的用户):**
*   **API：** 全称 Application Programming Interface，翻译为“应用程序编程接口”。您可以把它想象成软件和快递100服务器之间的一个“传话员”。您把单号给软件，软件让“传话员”去问服务器，然后“传话员”把结果带回来。

完成此步后，您的软件就可以正常查询快递了！

##### **1.5 第四步：查询并添加您的第一个快递**

1.  点击顶部的标签页，切换到 **“🔍 快递查询”**。
2.  在“快递单号”输入框里，输入一个您的快递单号（例如 `YT1234567890`）。
3.  “快递公司”一栏，建议您**直接保持默认的“自动识别”**。软件非常聪明，大部分情况下能根据单号格式自动判断是哪家快递公司。
4.  点击右边的 **“🔍 开始查询”** 按钮。
5.  稍等片刻，如果查询成功，软件会弹出“添加成功”的提示。此时，这个快递就已经被添加到您的“快递汇总”中了！

##### **1.6 第五步：管理您的所有快递（快递汇总页）**

这是您最常用的页面。切换到 **“📦 快递汇总”** 标签页，您添加的所有快递都会在这里按状态分类显示。

*   **查看状态：** 您会看到“在途中”、“派件中”、“已签收”等不同的分组，每个快递卡片上都显示了最新的物流信息。
*   **添加快递：**
    *   **手动添加：** 您可以在汇总页顶部的输入框中手动输入单号，点击“➕ 添加到汇总”来直接添加，无需切换到查询页。
    *   **批量导入：** 如果您有多个单号，可以点击 **“📦 批量导入”**。您可以复制粘贴多行文本，或者选择一个包含多个单号的txt文件，软件会一次性将它们全部加入监控列表。这对于代购、小商家来说非常方便！
*   **添加备注和截图：** 点击快递卡片上的“点击添加备注...”，您可以写上“给女儿买的粉色连衣裙”之类的备注。点击卡片左侧的图片区域，可以为这个快递添加一张商品截图，这样您就能把快递和商品对应起来了。
*   **刷新状态：** 您可以点击某个快递卡片上的“刷新”按钮来更新单个快递的状态，也可以点击顶部的 **“🔄 刷新全部”** 来一次性更新所有快递的状态。

#### **第二章：进阶应用篇（小商家、代购、行政前台必读）**

当您管理的快递数量增多时，掌握本章节的技巧可以极大提升效率。

##### **2.1 高效批量操作**

*   **批量刷新：** 除了“刷新全部”，您还可以点击任何一个分类标题旁的“刷新此类”按钮，例如只刷新“派件中”的快递。
*   **批量导出：** 点击“📋 批量导出单号”，会弹出一个选择窗口。您可以按分类、按关键词（例如“圆通”）筛选，然后选择需要的单号，导出为以下格式：
    *   **`.txt`：** 适合复制到记事本，或作为其他脚本的输入。
    *   **`.csv`：** 推荐！可以用 Excel 或 WPS 打开，形成一个包含单号、公司、状态、备注的表格，非常适合做账、对账、记录客户信息。
    *   **`.json`：** 适合有编程能力的用户进行二次开发。

##### **2.2 快递分类与状态管理**

软件会自动将快递分为十类，理解这些分类有助于您更好地管理订单：

*   **⏳ 待揽件：** 您添加了单号，但商家还没发货或快递员还没取件。
*   **📮 已揽收：** 快递员已取走包裹。
*   **✈️ 在途中：** 包裹正在运输的漫长旅途中。
*   **🚚 派件中：** 包裹已经到达目的地，快递员正在派送，这是您需要留意电话的时候。
*   **📍 已到驿站：** 包裹已放入菜鸟驿站、丰巢快递柜或代收点，您需要尽快凭取件码去拿。
*   **✅ 已签收：** 包裹已被签收。
*   **⚠️ 疑难件：** 快递出现了问题，如地址不详、电话联系不上等，需要您主动联系快递公司。
*   **↩️ 退件：** 包裹正在退回给寄件人。
*   **🛃 清关中：** 国际包裹正在海关接受检查。
*   **📦 其他：** 一些特殊或未知的状态。

**小技巧：** 您可以点击分类标题左侧的 **“▼”** 来折叠或展开该分类，让界面更清爽。

##### **2.3 强大的右键菜单**

在“快递汇总”页的任何一个快递卡片上，**单击鼠标右键**，您会发现一个新世界！根据您点击的位置不同（状态文字、备注按钮、物流按钮），弹出的菜单内容也不同。

*   **右键点击状态文字：**
    *   **复制单号/公司/状态：** 一键复制信息，方便粘贴给客户。
    *   **复制完整信息：** 一次性复制该快递的所有关键信息。
*   **右键点击备注按钮：**
    *   **添加快捷备注：** 预设了“衣服”、“电子产品”等常用词，点击即可快速备注。
    *   **复制备注：** 方便分享给同事。
*   **右键点击物流按钮：**
    *   **复制物流信息：** 复制最新的一条物流状态。
    *   **在网上搜索此单号：** 将用浏览器打开快递100官网并查询此单号，用于核验或分享给没有本软件的客户。

##### **2.4 多用户协同工作**

如第一章所述，您可以利用多用户系统为团队中的每个成员创建独立账号。例如：
*   **用户A (客服)：** 只负责添加客户单号和查看物流状态。
*   **用户B (售后)：** 专门处理“疑难件”分类下的快递。

每个用户的浏览历史、备注、截图数据是完全独立的。

#### **第三章：高级专家篇（开发者、技术爱好者、数据分析者必读）**

本章节深入软件的技术内核，揭示其强大的数据处理能力。

##### **3.1 本地智能大脑：送达时间预估与节点分析**

您是否好奇“从广州到北京，圆通快递平均要多久？”或者“我的包裹在‘广州转运中心’通常要停留几个小时？” 本软件不仅仅是一个查询工具，更是一个**本地物流数据分析平台**。

**它是如何工作的？**
1.  **数据收集：** 每当您的一个快递“已签收”或“已到达驿站”，软件会在后台悄悄地分析它的整个物流轨迹。
2.  **节点提取：** 软件会解析出包裹经过的每一个站点（例如`【广州转运中心】`）和对应的到达、离开时间。
3.  **数据入库：** 这些信息会被匿名化处理（只保留公司、城市、站点、耗时），存入您本地的 `delivery_history` 和 `node_transit_history` 数据库表中。
4.  **模型学习：** 随着您查询的快递越来越多，这个本地数据库就成了一个为您量身定制的物流时效预测模型。

**带来的好处：**
*   **精准的送达预估：** 当您查询一个新快递时，软件会基于该快递公司和目的地城市的历史数据，给出一个“预计X月X日送达”的提示。
*   **站点停留时间预估：** 在快递详情页的“站点时间轴”和“后续站点预估”中，软件会告诉您“基于历史数据，包裹在该站点预计停留X小时”或“预计X小时后到达下一站”，让您对物流节奏了如指掌。

**如何在全新电脑上利用此功能？**
*   您可以使用软件提供的 **“数据库备份与恢复”** 功能，将旧电脑上的数据文件迁移过来，您的“智能大脑”也就跟着过来了。
*   或者，您可以使用 **“数据库管理”** 页中的 **“全部重建”** 功能，如果您从其他途径获取了包含完整物流轨迹的 `express_summary` 数据，可以通过此功能快速生成本地历史数据库。

##### **3.2 API多账号管理与智能切换**

对于查询量非常大的用户（如企业用户），单个免费API账号的每日查询额度可能不够用。

*   **多账号配置：** 您可以在“API账号管理”中添加多个API Key。
*   **智能切换：** 当软件检测到当前账号的今日剩余额度为0时，它会**自动、平滑地切换到下一个可用账号**，整个过程对您的操作几乎是透明的，确保了查询服务的连续性。
*   **导入/导出：** 您可以轻松地将一套配置好的账号信息导出为加密或不加密的JSON文件，方便在团队内部署。

##### **3.3 数据持久化与安全：数据库管理与备份**

您所有的快递数据都安全地存储在 `data` 文件夹下的 `user_[您的用户名].db` 文件中，这是一个标准的 **SQLite3** 数据库文件。

*   **WAL模式：** 软件启用了SQLite的WAL (Write-Ahead Logging) 模式，这使得您可以一边查询一边写入，互不干扰，并极大地提升了并发性能。
*   **自动备份：** 在“设置”中可以开启“自动备份（每小时）”。软件会在 `data/backups` 目录下自动生成带时间戳的数据库备份文件，并自动清理超出设定数量的旧备份，确保您的数据万无一失。
*   **手动管理与恢复：** 点击工具栏的 **“🗄️ 数据库管理”** 标签页，再点击 **“💾 备份管理”**。
    *   **备份管理：** 您可以看到所有历史备份，并随时将数据库恢复到任何一个备份点。
    *   **优化数据库：** 点击“🔧 优化数据库”可以对数据库进行碎片整理，提升运行效率。
    *   **历史数据重建：** 这是一个高级功能。如果您导入了大量的历史物流数据，可以在这里点击“全部重建”，系统将重新扫描所有快递轨迹，为您生成本地的时效和节点历史数据。

##### **3.4 开发者接口与扩展性**

*   **数据可访问性：** 由于数据存储为标准的 SQLite3 格式，任何第三方SQLite客户端（如 DB Browser for SQLite, SQLiteStudio）都可以直接打开 `.db` 文件进行数据查询、分析和二次开发。
*   **导出格式：** 软件支持将历史记录、快递单号等导出为通用的 `CSV` 和 `JSON` 格式，方便与其它系统（如Excel、Python脚本、RPA机器人）进行数据交换。

### **第三部分：常见问题与故障排查 (FAQ)**

**Q1：查询快递时提示“未配置API账号”怎么办？**
**A1：** 请务必按照本指南 **第一部分 1.4节** 的步骤，去快递100官网申请免费的API Key，并添加到软件设置中。

**Q2：提示“查询失败”或“所有账号额度都已用完”怎么办？**
**A2：**
*   **额度用完：** 免费API账号有每日查询次数限制。如果是个人使用，请等待次日0点自动重置。如果是企业用户，建议申请付费套餐或配置多个账号以实现智能切换。
*   **其他错误：** 可能是网络问题或单号格式错误。您可以尝试右键点击物流信息，选择“在网上搜索此单号”来交叉验证。

**Q3：软件会偷偷上传我的快递数据吗？**
**A3：** **绝对不会。** 本软件是 **100% 本地运行**的工具。所有的快递单号、物流信息、备注、截图和历史数据都只存储在您的电脑上，唯一的对外网络通信就是使用您自己提供的API Key去快递100的服务器查询物流轨迹。软件作者及任何第三方都无法获取您的个人数据。

**Q4：我想换一台新电脑，怎么把旧电脑上的所有快递数据迁移过去？**
**A4：**
1.  在旧电脑上，找到软件所在的文件夹，将整个 **`data`** 文件夹复制到U盘。
2.  在新电脑上下载并解压本软件。
3.  将U盘中的 **`data`** 文件夹，覆盖粘贴到新电脑的软件根目录下即可。再次打开软件，您会发现所有的用户、快递记录、设置都原封不动地迁移过来了。

**Q5：为什么有些单号能自动识别，有些不能？**
**A5：** 软件的自动识别依赖于内置的单号格式规则库。主流快递公司（顺丰、三通一达、京东、邮政等）的单号格式都是固定的，因此识别率很高。但对于一些小的、不知名的快递公司，或者遇到新型号段，就可能无法识别。此时您可以手动在下拉框中选择正确的快递公司。

**Q6：我想修改快递的分类（比如把一个“在途中”的改成“疑难件”），可以吗？**
**A6：** 目前的版本中，分类是由软件根据物流状态和内容**自动判断**的，不允许手动修改。这是为了保持状态管理的客观性和一致性。如果您认为某个分类有误，那通常是快递公司上报的状态有问题，您可以尝试刷新一下，或者作为Bug提交到GitHub。
