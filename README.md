## 2. 什么是快递查询系统？

### 2.1 用大白话说

假设您在网上买了10件东西，有顺丰的、圆通的、中通的、京东的...每天要打开好几个App或者网站，一个一个输入单号查看物流，是不是很麻烦？

**Express_Query 就是解决这个问题的。**

您只需要：
1. 把快递单号输入到软件里（或者一次性导入）
2. 软件自动帮您查询
3. 所有快递的状态一目了然

### 2.2 一个生动的例子

**场景：** 小王是一个网购达人，双十一买了30件商品。

**没有用 Express_Query 的时候：**
- 每天打开淘宝App，找到"我的订单"
- 点开第一个订单，查看物流
- 返回，点开第二个订单...
- 反复30次，耗时20分钟
- 第二天忘记了哪些已经签收，又要重新查一遍

**用了 Express_Query 之后：**
- 把30个单号一次性导入软件
- 打开软件，所有快递状态一目了然
- "已签收"的快递自动归类在一起
- "派件中"的快递知道今天会到
- "待揽件"的快递知道商家还没发货
- 全程只需2分钟

### 2.3 软件的工作原理（简单版）

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   您输入    │     │   软件发送  │     │   返回结果  │
│   快递单号  │ ──→ │   到快递100 │ ──→ │   显示给您  │
│             │     │   API服务器 │     │             │
└─────────────┘     └─────────────┘     └─────────────┘
                            │
                    ┌───────┴───────┐
                    │               │
               ┌────▼────┐    ┌────▼────┐
               │ 顺丰系统 │    │ 圆通系统 │   ...
               └─────────┘    └─────────┘
```

软件通过"快递100"这个中间平台，获取各个快递公司的物流信息。您需要配置一个API账号（免费注册即可）才能使用。

## 3. 我需要什么才能使用？

### 3.1 硬件要求（最低配置）

| 项目 | 最低要求 | 推荐配置 |
|------|----------|----------|
| 操作系统 | Windows 7 及以上 | Windows 10/11 |
| 内存 | 2GB | 4GB 或以上 |
| 硬盘空间 | 100MB | 500MB（用于存储数据） |
| 网络 | 能上网就行 | 稳定的宽带 |
| 显示器 | 1024×768 | 1920×1080 |

### 3.2 软件要求

- **Python环境**：如果下载的是源代码版本，需要安装Python 3.8+
- **或者**：下载打包好的exe版本，无需安装任何东西，双击就能用

### 3.3 需要的账号

**快递100 API账号（必需）**

这是查询快递所必需的。不用担心，注册很简单而且免费：

1. 打开浏览器，访问：https://www.kuaidi100.com/openapi/
2. 点击"免费注册"
3. 填写信息，完成注册
4. 登录后，进入"控制台"
5. 找到"Customer"和"Auth Key"（也叫授权密钥）
6. 把这两个信息填入软件中

**免费额度说明：**
- 新注册用户通常有每天100次免费查询额度
- 用完后第二天自动重置
- 如果需要更多，可以购买付费套餐
- 本软件支持配置多个账号，一个用完自动换下一个

### 3.4 下载与安装

#### 方式一：下载打包好的exe（推荐普通用户）

1. 访问项目地址：https://github.com/duma520/Express_Query
2. 点击右侧的"Releases"
3. 下载最新版本的 `Express_Query_vX.X.X.zip`
4. 解压到任意文件夹
5. 双击 `Express_Query.exe` 运行

#### 方式二：从源代码运行（适合开发者）

```bash
# 1. 克隆项目
git clone https://github.com/duma520/Express_Query.git
cd Express_Query

# 2. 安装依赖
pip install -r requirements.txt

# 3. 运行程序
python Express_Query.py
```

## 4. 5分钟快速上手指南

### 第一步：首次启动（1分钟）

双击运行软件后，您会看到"选择用户"对话框：

```
┌──────────────────────────────────────┐
│      快递查询系统 v1.1.45             │
│                                      │
│          请选择用户登录               │
│                                      │
│  ┌────────────────────────────┐     │
│  │ 默认用户                    │     │
│  └────────────────────────────┘     │
│                                      │
│  [添加用户] [修改用户名] [删除用户]   │
│                                      │
│      [登录]          [退出程序]       │
│                                      │
└──────────────────────────────────────┘
```

**操作：** 选择"默认用户"，点击"登录"。

### 第二步：配置API账号（2分钟）

首次使用会提示您配置API账号：

1. 点击右上角的"⚙️ 设置"按钮
2. 点击"管理API账号"
3. 点击"添加账号"
4. 填写信息：
   - 账号名称：随便填，比如"主账号"
   - Customer：从快递100获取的Customer ID
   - Auth Key：从快递100获取的密钥
   - 每日额度：保持默认100
5. 点击"保存"

### 第三步：查询第一个快递（2分钟）

1. 切换到"🔍 快递查询"标签页
2. 在"快递单号"输入框中输入您的快递单号（比如：SF1234567890）
3. 快递公司选择"自动识别"（或者手动选择对应的快递公司）
4. 点击"🔍 开始查询"按钮
5. 等待几秒钟，物流信息就会显示出来

**恭喜！** 您已经成功查询了第一个快递。这个快递会自动保存到"快递汇总"中。

### 第四步：查看快递汇总

切换到"📦 快递汇总"标签页，您会看到刚才查询的快递已经按照状态分类好了：

### 5.3 五个主标签页介绍

#### 📦 快递汇总（最常用的页面）

这是您管理所有快递的核心页面。所有添加的快递都会显示在这里，并自动按状态分类。

**工具栏按钮：**
| 按钮 | 功能 |
|------|------|
| 手动添加单号输入框 | 输入要添加的快递单号 |
| 快递公司下拉框 | 选择快递公司（可选"自动识别"） |
| ➕ 添加到汇总 | 查询并添加到汇总 |
| 📦 批量导入 | 一次性导入多个单号 |
| 🔄 刷新全部 | 刷新所有快递的最新状态 |
| 📤 导出汇总 | 导出为CSV文件 |

**分类区域：**
- ✅ 已签收 - 已经收到的快递
- 📍 已到驿站 - 已到达菜鸟驿站/丰巢等
- 🚚 派件中 - 快递员正在派送
- ✈️ 在途中 - 正在运输中
- 📮 已揽收 - 快递员已取件
- ⏳ 待揽件 - 等待快递员取件
- ⚠️ 疑难件 - 有问题的快递
- ↩️ 退件 - 退回的快递
- 🛃 清关中 - 海关清关中

## 10. 常见问题解答（FAQ）

### Q1: 软件需要付费吗？

**A:** 完全免费！Express_Query 是开源免费软件，采用 AGPL-3.0 许可证发布。您可以自由使用、修改和分享。

### Q2: 为什么我需要注册快递100账号？

**A:** 查询快递需要调用快递100的API接口，这是快递100的要求。每个账号每天有免费额度（通常100次），对于个人使用完全足够。

### Q3: 免费额度用完了怎么办？

**A:** 有几个选择：
1. 等到第二天0点，额度自动重置
2. 注册多个快递100账号，配置到软件的"多账号管理"中，软件会自动切换
3. 购买快递100的付费套餐（如果查询量特别大）

### Q4: 我的数据保存在哪里？

**A:** 所有数据都保存在您电脑本地，位置是：
- 软件目录下的 `data` 文件夹
- 用户数据在 `data/user_data/` 中
- 备份文件在 `data/backups/` 中

**不会上传到任何云端服务器**，您的隐私是安全的。

### Q5: 换电脑了怎么迁移数据？

**A:** 复制整个 `data` 文件夹到新电脑的软件目录下即可。

### Q6: 软件打不开怎么办？

**A:** 
1. 确保是从官方GitHub下载的
2. 尝试以管理员身份运行
3. 检查杀毒软件是否误报（添加信任）
4. 如果是源代码运行，确保Python环境正确

### Q7: 查询总是失败怎么办？

**A:**
1. 检查API账号是否正确配置
2. 检查网络连接
3. 查看配额信息页面，确认还有剩余次数
4. 尝试查询一个确定有效的单号测试

### Q8: 可以同时管理多个人的快递吗？

**A:** 可以！使用"多用户系统"：
1. 点击"切换用户"
2. 添加新用户（如"老公"、"老婆"）
3. 每个用户的数据完全独立

### Q9: 快递状态不准确怎么办？

**A:** 点击"刷新"按钮获取最新状态。快递信息的更新取决于快递公司的系统，有时会有延迟。

### Q10: 可以手机使用吗？

## 11. 批量导入快递单号

### 11.1 什么是批量导入？

如果您需要一次性添加大量快递单号（比如作为电商卖家，每天要处理几十上百个订单），一个个手动输入太慢了。批量导入功能可以让您一次性导入所有单号。

### 11.2 批量导入的三种方式

**方式一：从文件导入**
1. 点击"📦 批量导入"按钮
2. 选择"从文件导入"
3. 点击"浏览"选择文件
4. 支持格式：TXT、CSV、文本文件
5. 文件内容：每行一个快递单号

**方式二：粘贴多行文本**
1. 选择"粘贴多行文本"
2. 将单号复制粘贴到文本框中
3. 可以选择分隔符（换行、逗号、空格等）
4. 点击"解析单号"

**方式三：手动输入**
1. 选择"手动输入（每行一个）"
2. 在文本框中输入单号
3. 每行一个

### 11.3 批量导入示例

**示例1：纯单号列表（TXT文件）**
```
SF1234567890123
YT1234567890123
ZT1234567890123
JD1234567890123
```

**示例2：带公司名称的混合格式**
```
顺丰 SF1234567890123
圆通：YT1234567890123
中通快递 ZT1234567890123
京东 JD1234567890123
```

软件会自动识别其中的快递公司名称和单号。

**示例3：CSV文件**
```csv
订单号,快递单号,快递公司
ORD001,SF1234567890123,顺丰
ORD002,YT1234567890123,圆通
ORD003,ZT1234567890123,中通
```

### 11.4 智能识别功能

批量导入时，软件会：

1. **自动提取单号**：从混合文本中识别出快递单号
2. **自动识别快递公司**：根据单号格式或文本中的公司名称
3. **自动去重**：重复的单号只导入一次
4. **预览确认**：导入前可以预览所有单号，确认后再执行

### 11.5 导入结果

导入完成后，软件会显示：
- 成功导入多少个
- 失败多少个（及失败原因）
- 跳过了多少个重复的单号

## 12. 智能分类系统详解

### 12.1 分类规则

软件通过分析物流信息的内容，智能判断快递应该属于哪个分类。判断规则如下：

| 分类 | 触发条件（满足任一即可） |
|------|------------------------|
| **已签收** | 状态码为3/40/75/85；或包含"签收/已收/妥投"等关键词 |
| **已到驿站** | 包含"驿站/菜鸟/丰巢/快递柜/取件码"等关键词 |
| **派件中** | 状态码为5/15/39/41/43/45；或包含"派件/派送"等关键词 |
| **已揽收** | 状态码为1/12/13/16/37/38/71/72/73；或包含"揽收/收件"等关键词 |
| **待揽件** | 无物流信息或显示"等待揽件" |
| **在途中** | 状态码为0/7/14等；或包含"在途/运输/中转"等关键词 |
| **疑难件** | 状态码为2/11/19/35/36；或包含"疑难/问题/异常"等关键词 |
| **退件** | 状态码为4/6/42/90/98；或包含"退回/退件"等关键词 |
| **清关中** | 状态码为8/9/10/34；或包含"清关/海关"等关键词 |
| **其他** | 以上都不匹配时 |

### 12.2 优先级说明

分类判断有优先级顺序，从高到低：

1. **已签收** - 最高优先级，因为签收是最终状态
2. **已到驿站** - 次高优先级，因为到驿站后通常很快会取件
3. **派件中** - 正在派送
4. **疑难件** - 出现问题需要关注
5. **退件** - 退回的快递
6. **清关中** - 国际快递
7. **已揽收/待揽件** - 初始状态
8. **在途中** - 默认状态
9. **其他** - 兜底分类

### 12.3 自定义分类（高级）

如果您想调整分类规则，可以修改源代码中的 `get_state_category` 方法。但需要注意：
- 修改后需要重新启动软件
- 建议先备份原文件

## 13. 送达时间预估功能

### 13.1 功能原理

送达时间预估功能基于**历史数据分析**：

1. 软件会记录您所有已签收的快递
2. 记录内容包括：快递公司、揽收时间、签收时间
3. 当有新的快递时，软件会查找同一快递公司的历史数据
4. 计算平均配送天数，给出预估

### 13.2 如何使用

预估是**自动进行**的，您无需任何操作。

在快递汇总中，每个快递条目会显示预估信息：
- "预计04月15日送达 (基于15条历史记录，该快递公司平均2.5天送达)"
- "派件中，预计今天或明天送达"
- "已到达取件点，请尽快取件"

### 13.3 提高预估准确度

预估的准确度取决于历史数据的数量：

| 历史数据量 | 准确度 | 说明 |
|-----------|--------|------|
| 0条 | 使用默认值 | 顺丰2天、京东2天、三通一达3天等 |
| 1-2条 | 较低 | 会显示"数据较少，仅供参考" |
| 3条以上 | 较高 | 基于实际平均数据 |
| 10条以上 | 很高 | 数据越多越准确 |

### 13.4 管理时效数据

在"设置 → 时效数据管理"中：
- 可以查看已记录的数据量
- 可以清空历史数据（重新开始学习）

## 14. 多账号管理与自动切换

### 14.1 为什么需要多个账号？

快递100的免费API账号每天有查询次数限制（通常100次）。如果您：
- 管理大量快递
- 频繁刷新状态
- 多人共享使用

100次可能不够用。配置多个账号可以：
- 每个账号100次/天，2个就是200次
- 一个账号用完自动切换到下一个
- 最大化利用免费额度

### 14.2 如何添加多个账号？

1. 点击"⚙️ 设置" → "管理API账号"
2. 点击"添加账号"

## 28. 故障排查指南

### 28.1 常见错误及解决

| 错误现象 | 可能原因 | 解决方法 |
|----------|----------|----------|
| 软件无法启动 | 缺少运行库 | 安装VC++运行库 |
| 界面显示异常 | 显卡驱动问题 | 更新显卡驱动 |
| 数据库连接失败 | 文件权限问题 | 以管理员身份运行 |
| 查询超时 | 网络问题 | 检查防火墙/代理设置 |
| API返回错误 | 账号配置错误 | 重新配置API账号 |

### 28.2 数据恢复

如果数据库损坏：
1. 找到 `data/backups/` 目录
2. 选择最近的备份
3. 使用备份管理功能恢复

### 28.3 完全重置

如果需要完全重置软件：
1. 关闭软件
2. 删除 `data` 文件夹
3. 重新启动软件

| 复制功能 | 基础 | 增强版 |
| 界面显示 | 基础 | 优化显示 |

### 30.2 新增功能详解

#### 1. 批量导入功能（全新）

**v1.1.34：** 只能一个一个手动添加

**v1.1.45：** 支持三种导入方式
- 从文件导入（TXT、CSV）
- 粘贴多行文本
- 手动输入

**使用场景：**
```
之前：30个快递需要30次操作，每次输入单号→选择公司→点击添加
现在：30个快递1次操作，粘贴所有单号→解析→一键导入
效率提升：约30倍
```

#### 2. 快递公司识别（大幅扩展）

**v1.1.34：** 仅支持13家
```
顺丰、圆通、中通、韵达、申通、京东、邮政、百世、天天、德邦、极兔、EMS
```

**v1.1.45：** 支持50+家，新增包括：
```
国内：宅急送、优速、跨越、速尔、全峰、国通、快捷、龙邦、信丰、增益、
      联昊通、全一、如风达、安能、壹米滴答、德坤、中铁、民航、菜鸟、丹鸟
国际：联邦快递、DHL、UPS、TNT、USPS、亚马逊物流
```

#### 3. 账号切换智能重试机制（核心优化）

**v1.1.34逻辑：**
```
查询失败 → 尝试切换账号 → 不管成功与否，结束
```

**v1.1.45逻辑：**
```
查询失败 → 记录失败原因
         ↓
    强制切换到下一账号
         ↓
    询问用户是否重试
         ↓
    自动用新账号重新查询
         ↓
    成功则更新，失败则继续尝试
```

**批量刷新重试：**
```
v1.1.34：批量刷新中某个失败，只能手动处理
v1.1.45：自动收集失败单号 → 切换账号 → 批量重试失败项
```

#### 4. 右键菜单全面增强

| 位置 | v1.1.34 | v1.1.45 |
|------|---------|---------|
| 状态标签 | 无 | 复制单号/公司/状态/完整信息 |
| 备注按钮 | 无 | 复制备注/快捷备注/编辑 |
| 物流按钮 | 无 | 复制物流/查看详情/刷新/搜索 |
| 截图区域 | 基础菜单 | 查看/添加/编辑/导出/复制/信息 |

#### 5. 截图功能大幅增强

| 功能 | v1.1.34 | v1.1.45 |
|------|---------|---------|
| 添加方式 | 文件、剪贴板 | 文件、剪贴板、URL下载 |
| 编辑功能 | 无 | 旋转（90°/180°）、裁剪 |
| 导出功能 | 无 | 导出原图、导出带信息 |
| 复制功能 | 无 | 复制到剪贴板 |
| 信息查看 | 无 | 查看尺寸、大小 |

#### 6. 账号导入导出（新增）

**v1.1.34：** 不支持，换电脑需手动重新配置

**v1.1.45：**
